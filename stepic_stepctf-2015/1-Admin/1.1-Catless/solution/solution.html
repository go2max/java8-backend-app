При подключении к контейнеру участник соревнований попадает в систему, в которой удалены основные утилиты для редактирования и вывода текстовых (и не только) файлов, т.е. такие программы, как cat, less, more, tail, head, etc. Участнику необходимо найти способ прочитать файл flag.txt. Для этого можно воспользоваться не такими распространенными утилитами, как nl (выводит файл с пронумерованными строками), cut (позволяет вырезать куски из файлов, например, сработает 'cut -b 1- flag.txt'), tac (выводит файл задом наперед), или же можно прочитать файл прямо из bash: while read line do printf "$line" done < flag.txt Вместо printf можно использовать и echo, но для этого сначала нужно сбросить псевдоним для echo ($ unalias echo), который коварно указывает на несуществующий /bin/echo. Также, можно переименовать файл в C-файл ($ cp flag.txt flag.c) после чего скормить его компилятору ($ gcc flag.c), ответ будет выдан с текстом ошибок. cp -f flag.txt /proc/self/fd/1 cp запишет в файл, открытый у нашего процесса(bash) как файловый дескриптор 1, а это терминал. так же можно писать в /dev/stdout или в один из /dev/ttyx или /dev/pts/x, если знать свой Answer: STCTF#l1nvxH4sN1ceStvff#
